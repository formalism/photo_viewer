# 画像ファイルビューア 実装報告書

Spec.mdに基づき、以下の機能を実装しました。

## 実装済み機能

### 1. Google認証 (OAuth2)
- **認証フロー:** `remix-auth` および `remix-auth-oauth2` (`OAuth2Strategy`) を使用し、GoogleのOAuth2エンドポイントを直接構成して実装しました。
- **認可ロジック:** 
  - ログイン可能なユーザーはSQLiteデータベースで管理されます。
  - **初期状態:** データベースが空の場合、どのGoogleアカウントでもログイン可能です（初期セットアップ用）。
  - **運用状態:** 1人以上のユーザーが登録された後は、登録済みユーザーのみがログインおよび管理画面の利用が可能になります。

### 2. データベース・管理画面 (`/db`)
- **ORM:** `Drizzle ORM` を採用し、タイプセーフなデータベース操作を実現しました。
- **ユーザー管理:** 利用許可を出すGoogleメールアドレスの登録・削除が可能です。
- **マッピング管理:** URLパス（例: `/photos`）とサーバー上のディレクトリ絶対パス（例: `/var/data/images`）の紐付け設定が可能です。

### 3. 画像/動画ビューア
- **ディレクトリブラウジング:** 設定されたURLパスにアクセスすると、ディレクトリ内のファイルとフォルダをグリッド表示します。
- **自動サムネイル生成:**
  - `sharp` を使用した画像のサムネイル生成（横幅320px、アスペクト比維持）。
  - `fluent-ffmpeg` を使用した動画のサムネイル生成（開始2秒時点のフレームを抽出）。
  - 生成されたサムネイルは、対象ディレクトリ内の `.thumbs` フォルダにキャッシュされます。
- **オリジナルファイル配信:** サムネイルをクリックすると、適切な権限チェックを経た上で、オリジナルファイルを直接（HTML埋め込みなしで）表示・再生します。

### 4. セキュリティ
- 全てのビューアアクセスおよびファイル配信において、セッション確認とユーザー許可のチェックを強制しています。
- パス解析において `..` などの記号をチェックし、指定されたディレクトリ外へのアクセス（ディレクトリトラバーサル）を防止しています。

## 技術スタック
- **Framework:** React Router v7 (Framework mode)
- **Database:** SQLite (better-sqlite3) via **Drizzle ORM**
- **Image Processing:** sharp
- **Video Processing:** fluent-ffmpeg (ffmpeg 6.1.1 導入済み)
- **Authentication:** remix-auth, **remix-auth-oauth2**

## セットアップ手順

1.  **.env ファイルの設定**
    `.env.example` をコピーして `.env` を作成し、Google Cloud Consoleで取得した `GOOGLE_CLIENT_ID` と `GOOGLE_CLIENT_SECRET` を設定してください。
    ```env
    GOOGLE_CLIENT_ID=あなたのクライアントID
    GOOGLE_CLIENT_SECRET=あなたのクライアントシークレット
    SESSION_SECRET=ランダムな文字列
    ```

2.  **依存関係のインストール**
    ```bash
    npm install
    ```

3.  **データベースのセットアップ**
    ```bash
    npx drizzle-kit push
    ```

4.  **起動**
    ```bash
    npm run dev
    ```

5.  **初期設定**
    - `http://localhost:3000/db` にアクセスし、最初のユーザーを登録してください。
    - その後、表示したいディレクトリのマッピングを追加することで、指定したURLパスから画像を閲覧できるようになります。
