# 画像ファイルビューア

## 機能
- Googleアカウントでログインできる (未認証ユーザは利用不可)
    - OAuth2で認証
    - ログインできるGoogleアカウントは、SQLiteで管理する。
    - /dbにアクセスすると、アカウント管理画面を表示する
        - /dbというパスだけは特別扱いとする
        - DBが空の場合は、誰でもログインできる
        - 一旦DBに1アカウントでも設定された後は、設定済みアカウントのみがアカウント管理画面を利用できる(登録されたアカウントは誰でも管理画面を利用可能とする)
- 指定されたディレクトリにある画像/動画をサムネイル表示する。symlinkは追わない。
    - ディレクトリ内にある全画像ファイルのサムネイルを、グリッド状に配置して表示する。
    - 当該ディレクトリが更に別のディレクトリを含む場合は、フォルダアイコンを表示する。
        - フォルダアイコンクリックで、さらに下のディレクトリの中を表示する。
- サムネイルをクリックすると、オリジナル画像/動画を表示する。
    - オリジナル画像/動画を表示する際は、オリジナルデータそのものを表示し、HTMLに埋め込んだりはしない(当然権限チェックは行う)。
    - 動画のサムネイルは、動画開始2秒の時点の映像を利用する。動画が2秒に満たない場合は、任意のフレームを利用する。

## Tech Stack
- [React Router](https://reactrouter.com/) フレームワークモードで利用
- [SQLite](https://sqlite.org/) 利用者、URLパス<->ディレクトリ対応管理
- [sharp](https://www.npmjs.com/package/sharp) 画像のサムネイルを作成
- [FFmpeg](https://www.ffmpeg.org/) 動画からサムネイルを作成するために利用。デプロイするサーバにインストールされている前提

## Deployment

ベアメタルサーバ上のNode.jsで動かすものとする。

## 設計

- 作成したサムネイルは、当該ディレクトリの.thumbsというディレクトリの下に置いておく。
    - 例：/a/img.jpg に対しては、/a/.thumbs/img.jpgという名前で作成される。
        - 画像のアスペクト比を保つ。横幅が320ピクセル程度になるようにする。
    - 例：動画の場合。/a/mov.mpg に対しては、/a/.thumbs/mov.jpgという名前で静止画サムネイルが作成される。
- 特定のディレクトリを見ようとする時には、まず.thumbsを参照して、サムネイルがあるか確認する。
    - サムネイルが存在する場合は、それをファイル一覧に表示する。
    - サムネイルが存在しない場合は、作成してからファイル一覧に表示する。

### データベース

/db というパスは特別扱いし、以下のテーブルの情報を設定できるものとする。

- 利用者テーブル: このテーブルに登録されたユーザのみがディレクトリを閲覧可能
    - id: pkey, unsigned int
    - email: varchar(64) not null. Googleのメールアドレス。
- ユーザ, パス, ディレクトリ マッピングテーブル:
    - id: pkey, unsigned int
    - user_id: 利用者テーブルのidへの外部キー
    - url_path: not null. URLパス(例：/photos)。利用者ごとにパスとディレクトリの対応は異なって良いので、unique制約はつけない。
    - directory: not null. 表示するディレクトリ(絶対パス)。
